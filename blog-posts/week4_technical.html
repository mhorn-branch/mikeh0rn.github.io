Week 4 Technical Blog: Enumerable#map

September 21, 2014

One of the key methods we dove into this week was the Enumerable#map (.map) method. I had done some research on this method because I was particular intrigued by how this method differentiated itself from a method like ".each" since they seemed to be applied in the same fashion. However, the key takeaway from this week is that map is used to, in short, to return a new array after items have been iterated over in a block of code vs. .each which will just return the original array. From a non-technical standpoint, it will look at the items in a group of numbers or items (the array) and say "For each one of these items, do this to it. And when you're done doing what I told to do to this, give me a brand new version of the original thing but with the items in the same order."

If this were attempted using .each, it would ultimately spit back out the same array you gave it. Each is used to process an array but does not have the power to create a new array with manipulated items.

An example using each... [1,2,3].each { |i| i * 2 } #=> [1,2,3]

Note: you're getting the SAME array back. Hmm I guess we should try map to give us that new array we were looking for to apply that block of code over each item in the array...

[1,2,3].map { |i| i * 2 } #=> [2,4,6]
