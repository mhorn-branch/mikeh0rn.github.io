9/28/14: Ruby Classes

Simply put, like most things in Ruby, Classes are objects. They are models to organize data to create new objects. This may sound long-winded and confusing but as you'll in the example below, Classes are really great structures for more in-depth programming in Ruby. It's a great way of keeping items organized and making sure certain methods we create can only be used within that Class. Similar to what I did for my JavaScript solo challenge, I'll stick to the music theme. Here's an example where we initialize a parameter that takes a band name and a band album and once they are entered, it'll return an interpolated string when we create a new instance of that Class. If you're confused well so am I so don't worry. I tried to make a very basic class to spell out the approach. Sorry for the lack of indentation. This is will be fixed throughout the next few weeks as I continue to update my blog layout...

class Music
def initialize (bandname, favalbum)
@bandname = bandname
@favalbum = favalbum
end

def bands
return "I really love the band #{@bandname}! Especially, the album #{@favalbum}."
end
end

With every class, you need to include an initialize method for the parameters it's going to take. This is ultimately the guts of the Class that will allow you to add instance variables that could be defined and used later within the class at any point in the Class. The beautiful thing about instance variables (items with an '@' symbol) is that they are available to you throughout the class in any way, shape or form. Through the initialize method, you state your instance variables and using the sugar syntax that ruby makes so beautiful, you can do things like @bandname = bandname and when you pass the argument through the initialize parameters, the string interpolation within the bands method will allow that instance variable to be referenced. When Led Zeppelin is entered, they'll be entered through the instance variable @bandname within the string.
